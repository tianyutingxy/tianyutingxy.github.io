      OpenCV初体验

      Post on 2015-09-16

      前言

      OpenAV是一个跨平台且高效的图像处理库。

      正文

      原本研究视频拍摄实时滤镜，研究研究着方向就歪了，看了一些OpenCV处理图像的效果，觉得非常厉害，写了个Demo给大家介绍下。

      有个APP叫“扫描全能王”，用这个app拍摄完图片后会自动识别文件的边缘，并把文件从照片上抠下来拉伸为标准的纸张大小，如下所示。


      实现的原理就是透视变换(perspective transform)，OpenCV是可以直接实现。我们就从这个效果开始了解OpenCV。

      OpenCV封装了一批常用的图片操作函数，并封装到了Imgproc这个类中。

      例如包含获取图片直方信息、边缘检测、马赛克、霍夫变换等，我们摘几个函数演示一下：

      switch (mProcessMethod) {
        case Canny:
            mRgba = inputFrame.rgba();
            Imgproc.Canny(inputFrame.gray(), mTmp, 80, 100);
            Imgproc.cvtColor(mTmp, mRgba, Imgproc.COLOR_GRAY2RGBA, 4);
            break;
        case HoughLines:
            Imgproc.Canny(mRgba, mIntermediateMat, 80, 100);
            rgbaInnerWindow = mRgba.submat(top, top + height, left, left + width);
            Imgproc.cvtColor(mIntermediateMat, rgbaInnerWindow, Imgproc.COLOR_GRAY2BGRA, 4);
            Mat lines = new Mat();
            Imgproc.HoughLines(mIntermediateMat, lines, 1, Math.PI / 180, 150);

            if (lines.cols() > 0) {
                double[] data;
                double rho, theta;
                Point pt1 = new Point();
                Point pt2 = new Point();
                double a, b;
                double x0, y0;
                for (int i = 0; i < lines.cols(); i++) {
                    data = lines.get(0, i);
                    rho = data[0];
                    theta = data[1];
                    a = Math.cos(theta);
                    b = Math.sin(theta);
                    x0 = a * rho;
                    y0 = b * rho;
                    pt1.x = Math.round(x0 + 1000 * (-b));
                    pt1.y = Math.round(y0 + 1000 * a);
                    pt2.x = Math.round(x0 - 1000 * (-b));
                    pt2.y = Math.round(y0 - 1000 * a);
                    Imgproc.line(mRgba, pt1, pt2, mColorsRGB[1], 3);
                }
            }
            break;
        default:
            mRgba = inputFrame.rgba();
            break;
    }
      • Canny（边缘检测）

      边缘检测是实时的计算速度非常快，通过边缘检测的图片信息保存下来的数据可以用作下一步霍夫变换的值线检测。

      • HoughLines（霍夫变换）

      屏幕的帧率掉的非常厉害，从16fps直接跌到了6fps，实时的计算是非常慢的，而且由于限制了计算的阈值，居然只算出一条直线。。。

      最好是采集图像完毕后，用单张图去做霍夫变换。

      结尾

      1. OpenCV处理旋转非常山寨。
      2. OpenCV处理全屏非常山寨。
      3. OpenCV处理灰度、马赛克、放大镜非常厉害。
