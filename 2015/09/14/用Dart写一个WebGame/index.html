      【web】用Dart写一个WebGame

      Post on 2015-09-14

      在Dart DEV的峰会上有一段介绍使用dart写游戏的视频，看完觉得碉堡了。

      我们就来探索一下如何使用dart编写一个基本的web类游戏，从codeproject上找到一个sample。

      这是游戏的效果：

      作者已经把代码托管到了jsfiddle上，大家可以直接去玩儿了

      我们这里大致分析一下，在dart编写游戏的流程。

      根据我们过往开发游戏的流程，我们在一个新的平台上开发游戏大致关心三点：

      • 渲染（2d&3d）
      • gameloop
      • 库(输入输出、audio…)

      这些条件满足了就可以写游戏了。

      我们先看dart的库，这里我们关心两个，一个是keyboard一个是audio，用于接收用户数据与播放游戏音乐。

      Keyboard

      part of lostsouls;

class Keyboard {
  final HashSet<int> _keys = new HashSet<int>();

  Keyboard() {
    window.onKeyDown.listen((KeyboardEvent e) {
        _keys.add(e.keyCode);
    });

    window.onKeyUp.listen((KeyboardEvent e) {
      _keys.remove(e.keyCode);
    });
  }

  isPressed(int keyCode) => _keys.contains(keyCode);
}

      这里的window对象来自于

      import 'dart:html';

      这个库基本封装了对html dom的操作，也包含监听键盘的输入，所以够这个demo用了。

      Audio

      import 'dart:web_audio';

      这个库包含了音频的解码播放之类的。

      游戏demo在初始化的时候同时初始化了一个StateLoad类：

      GameHost(this.canvas) {
    keyboard = new Keyboard();
    var renderer = new Renderer(canvas.context2D, canvas.width, canvas.height);
    currentState = new StateLoad(keyboard, renderer);
  }

      这个类用于控制游戏状态以及音频资源：

      StateLoad(final Keyboard keyboard, final Renderer renderer) : super(keyboard, renderer, null) {

      这里只是指定了音频资源的url，并没有加载数据到内存中，真正的加载是异步的。在项目封装的AudioManager中，在AudioManger的构造函数中开始异步加载这些音频资源。

      void _loadBuffers() {
    for (var i = 0; i < _clips.length; i++) {
      _clips[i]._context = _context;
      _loadBuffer(_clips[i], i);
    }
  }

  void _loadBuffer(final SoundClip clip, final int index) {
    final HttpRequest request = new HttpRequest();
    request.open("GET", clip._url, async: true);
    request.responseType = "arraybuffer";
    request.onLoadEnd.listen((e) => _onBufferLoaded(request, clip, index));
    request.onError.listen((e) => _handleError(e, clip));

    try {
      request.send();
    }
    catch(ex) {
      clip._broken = true;
      _notify(clip);
    }
  }

      这个写法简直和C#一样,写起来真是顺顺的。|_・)

      这样游戏demo关于音频这块也够用了。

      ps.附一张游戏状态🐔

      接下来分析游戏主逻辑。一个是游戏循环，一个是页面渲染：

      gameloop

      游戏内的循环，依旧是调用window（dart:html）提供的方法。

      void requestRedraw() {
   window.requestAnimationFrame(draw);
 }

      我们需要的就是不停的自循环，同时处理逻辑

      void draw(num _) {
    final num time = new DateTime.now().millisecondsSinceEpoch;
    if (renderTime != null)
      showFps(_, 1000 / (time - renderTime));
    renderTime = time;

    double elapsed = 0.0;
    if (lastTimestamp != 0) {
      elapsed = (time - lastTimestamp) / 1000.0;
    }

    lastTimestamp = time;
    if (currentState != null) {
      var nextState = currentState._update(elapsed);
      currentState._render();
      if (currentState != nextState) {
        currentState = nextState;
        currentState._initialize();
      }
    }

    requestRedraw();
  }

      这里需要处理的逻辑有：

      • 读取输入数据
      • 计算上一帧距现在流逝的时间
      • 根据输入与流失时间更新游戏状态
      • 重绘页面
      • 开始下一轮循环

      好了，到这里游戏循环也够用了。

      render

      游戏里只用到了2d元素，主要是文本元素和多边形。使用的绘画工具也是系统提供的

      final CanvasRenderingContext2D context;

      所有游戏的sprite都是用这个画出来，所有绘画的方法也封装到了render.dart中：

      class Renderer {
  final CanvasRenderingContext2D context;
  final int _width;
  final int _height;
  double _previousGlobalAlpha = 1.0;

  *

  void fillPolygon(List<Vector2D> polygon, String fill) {
final Vector2D start = polygon[0];
context..fillStyle = fill
       ..beginPath()
       ..moveTo(start.x, start.y);

for(var i = 1; i < polygon.length; ++i) {
  final Vector2D point = polygon[i];
  context.lineTo(point.x, point.y);
    }

context..closePath()
       ..fill();
      }
}

      好了，现在所有元素具备，我们把dart编译成js就可以放到浏览器里运行了。满帧运行很流畅！而且好玩！
